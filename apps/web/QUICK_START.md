## 🚀 Get Started in 3 Steps

### Step 1: Set Up Environment (5 minutes)

```bash
cd /Users/anusha/Documents/LinerNotes/web

# Run setup script
chmod +x scripts/setup-and-test.sh
./scripts/setup-and-test.sh
```

This will:
- Check for `.env.local` and create template if missing
- Generate `NEXTAUTH_SECRET`
- Install dependencies
- Generate Prisma client

### Step 2: Configure Google OAuth (5 minutes)

Follow `GOOGLE_OAUTH_SETUP.md` to:
1. Create Google Cloud project
2. Enable Google+ API
3. Create OAuth client ID
4. Add credentials to `.env.local`

**Quick link:** https://console.cloud.google.com/

Your `.env.local` should have:
```bash
NEXTAUTH_URL=http://localhost:3000
NEXTAUTH_SECRET=<generated by script>
GOOGLE_CLIENT_ID=<from Google Console>
GOOGLE_CLIENT_SECRET=<from Google Console>
DATABASE_URL=<your Railway PostgreSQL URL>
```

### Step 3: Run Database Migration (1 minute)

```bash
# Connect to your Railway database and run migration
npx prisma migrate dev --name add_nextauth_models
```

## 🐛 Troubleshooting

### "Access blocked: This app's request is invalid"
- Check redirect URI in Google Console: `http://localhost:3000/api/auth/callback/google`
- Must match exactly (no trailing slash, http not https)

### "Database connection failed"
- Check `DATABASE_URL` in `.env.local`
- Ensure Railway database is running
- Test connection: `npx prisma db pull`

### "NEXTAUTH_SECRET is invalid"
- Must be 32+ characters
- Regenerate: `openssl rand -base64 32`

### "Session not persisting"
- Clear cookies and try again
- Check `NEXTAUTH_URL` matches your dev server URL
