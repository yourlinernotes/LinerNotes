#### 🔴 Spotify Refresh Token Expiration — July 20 deadline

**Announced June 18, 2026.** Existing apps (created before June 18) are affected from **July 20, 2026** — approximately 3 weeks away.

- Refresh tokens now have a hard **6-month lifetime** from the original authorization date.
- Refreshing an access token does **not** reset the clock.
- On expiry, `POST /api/token` returns `HTTP 400 {"error": "invalid_grant"}`.
- Only affects **Authorization Code** and **PKCE** flows — **not** Client Credentials.
- Source: https://developer.spotify.com/blog/2026-06-18-refresh-token-expiration

**Impact on this project:** `apps/web/src/lib/spotify.ts:97` throws a generic `"Failed to refresh access token"` on any `!response.ok`, without distinguishing `invalid_grant`. Users whose Spotify connection is older than 6 months will hit a broken state silently after July 20 — no re-auth prompt.

## Summary

### 🔴 Broken Now / Action Required

#### 1. Spotify refresh token expiration — apps/web (deadline: July 20, 2026)

**File:** `apps/web/src/lib/spotify.ts:97`

Current code throws a generic error for any `!response.ok` during token refresh — it cannot distinguish an `invalid_grant` (permanent expiry) from a transient error. After July 20, any user whose Spotify connection was authorized more than 6 months ago will hit a broken state with no recovery path.

**Suggested fix:**
```typescript
if (!response.ok) {
  const errorBody = await response.json().catch(() => ({}));
  if (errorBody?.error === 'invalid_grant') {
    // Refresh token expired (Spotify 6-month policy, enforced Jul 20 2026).
    // Caller must delete the stored connection and prompt re-auth.
    throw Object.assign(new Error('Spotify refresh token expired'), { code: 'SPOTIFY_TOKEN_EXPIRED' });
  }
  throw new Error(`Failed to refresh access token: ${response.status}`);
}
```

Then in the callers that use the stored connection: catch `SPOTIFY_TOKEN_EXPIRED`, delete the `MusicConnection` row, and return an appropriate error so the UI can show "Please reconnect Spotify."

Also consider storing `connectedAt` on `MusicConnection` and proactively prompting re-auth at the 5-month mark.

Source: https://developer.spotify.com/blog/2026-06-18-refresh-token-expiration
